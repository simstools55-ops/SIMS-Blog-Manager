# Product 5.0 Release 1 Sprint 5

## 目的

効果測定機能を Product 5.0 Official の4週間モデルへ完全統合し、旧メニュー・旧見出し・旧測定仕様が残る問題を解消する。

## 実装内容

- 改善履歴を7日・14日・21日・28日の4回測定形式へ強制移行
- スプレッドシートを開いた時にも正式スキーマを適用
- 「シートを作成・修復」実行時に正式スキーマを再適用
- 改善効果メニューから旧「7日間延長」「測定完了」を完全除去
- 改善履歴から旧「測定予定日」列を除去
- 旧測定予定日を実測日時へ誤移行しないよう修正
- メインメニューへ「バージョン情報」を追加
- バージョン表示を `5.0.0-official-r1-s5` に更新
- 配布用Apps Scriptを `apps-script/Code.gs` 一つに統一
- 旧分割ソース `src/` を配布パッケージから除外
- GitHub Pagesマニュアルを4週間測定仕様へ更新

## 既存データ

改善履歴の既存記事情報・改善内容・改善前数値は、見出し名を使って新しい列へ移行する。旧「測定予定日」は予定値であり実測値ではないため、1回目測定日時には移行しない。

## 適用手順

1. `apps-script/Code.gs` をApps Scriptの「コード.gs」へ全置換して保存
2. スプレッドシートを再読み込み
3. `SIMS-Blog-Manager → バージョン情報` で `5.0.0-official-r1-s5` を確認
4. `SIMS-Blog-Manager → シートを作成・修復` を1回実行
5. 改善履歴の4回測定見出しと改善効果操作メニューを確認
