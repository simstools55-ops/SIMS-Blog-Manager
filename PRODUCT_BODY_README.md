製品本体は `apps-script/Code.gs` です。

今回、記事DB直結の「今日の改善」と「改善ナビ」を追加しています。

# Product 5.0 ArticleDB Operation Menu Fix

記事DBの対象行を選択し、Google Sheets上部の独立メニュー「記事DB操作」から操作します。

## 記事DB操作

- 記事DBを開く
- 選択記事の詳細を開く
- 選択記事をブラウザで開く
- 改善ブリーフ（準備中）
- 効果測定（準備中）
- 改善完了（準備中）

セルプルダウンやインストール型編集トリガーは使用しません。

## SIMS Feedback Protocol v1

改善ナビからAIへ渡した記事改善依頼に対し、AIが回答末尾へ出力する `SIMS_FEEDBACK_V1` JSONをSIMSへ貼り付けて登録できます。利用者はJSONを編集する必要はありません。登録時に対象記事と変更内容を確認し、記事DB更新、改善履歴保存、モニター開始まで実行します。

## RC11

改善ナビのAI共通化、SIMS AI Protocol v1.1、改善履歴の可視化、モニター中記事の優先表示、改善中シート廃止を反映しています。


## RC11 起動時の記事DB更新確認

起動のたびに最終更新日時を表示し、記事DBを更新するか利用者が選択します。更新を見送っても次回起動時に再表示されます。手動更新は「SIMS-Blog-Manager」または「記事DB操作」メニューから実行できます。

主要メニューはサブメニューを使わず、1回のプルダウンから直接実行できる構成です。


## RC11 Selection Workflow

記事管理・今日の改善・効果測定・改善履歴を一択チェックボックスで操作できるようにし、利用順にシートとメニューを再編しました。
