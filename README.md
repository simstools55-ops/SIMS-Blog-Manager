# SIMS-Blog-Manager Product 4.2

Google Search Consoleのデータから、ブログ改善で今日やるべきことを提示するスプレッドシート型ツールです。

## Product 4.2の主な改善

- Search Consoleデータ取得成功後のUIを改善
- 「今日の改善」を見やすいタスクリスト形式に変更
- 通常表示はおすすめ5件に固定
- 必要な人向けに「改善候補をすべて表示」メニューを追加
- 改善ブリーフはシート内の長文セルではなく、別管理＋ダイアログ表示へ変更
- 接続エラー時にApps ScriptとGoogle Cloudのプロジェクト番号不一致を確認する案内を追加

## セットアップの注意

初回実行時はGoogleの承認画面が表示されます。これは正常な動作です。承認後、同じSTEPをもう一度実行してください。

Search Console APIを有効化しても接続できない場合は、Apps Scriptに設定されているGoogle Cloudプロジェクト番号と、APIを有効化したGoogle Cloudプロジェクト番号が一致しているか確認してください。

## 今日の改善の使い方

1. メニューから「今日のデータを取得・分析」を実行
2. 「今日の改善」シートを確認
3. 対象行を選択
4. メニュー「今日の改善」から以下を実行
   - 選択行の記事を開く
   - 選択行の改善ブリーフを開く
   - 選択行を完了にする

## 表示件数

既定ではおすすめ5件だけを表示します。
すべて見たい場合は、メニューから「改善候補をすべて表示する」を実行してください。
