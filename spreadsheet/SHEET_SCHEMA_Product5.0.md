# SIMS-Blog-Manager Product 5.0 Sheet Schema

## Home

毎日最初に見る画面。表示内容は以下の3点に限定する。

- ブログ全体の状況
- 改善作業の状況
- 今日やること

## 今日の改善

改善候補10件のうち初期2件を表示し、最大6件まで追加表示する。
改善完了後は改善ログへ記録し、改善中へ移動する。

## 改善中

改善実施日、経過日数、状態、SIMS評価、次のアクション、URL、修正内容、改善内容を管理する。
改善後7日・14日・21日・28日の4回測定対象とする。

## 改善ログ

改善履歴を保存する。改善前CTR、順位、クリック、表示回数を保持する。

## 処理ログ

Product 5.0では以下の列を正式仕様とする。

| 列 | 内容 |
|---|---|
| 開始時刻 | 処理開始日時 |
| 終了時刻 | 処理終了日時 |
| 利用者待ち時間 | 利用者が待った時間 |
| 処理 | 処理名 |
| 状態 | 完了またはエラー |
| 対象件数 | 処理対象件数 |
| 処理件数 | 実際に処理した件数 |
| 件数 | 表示用件数 |
| 詳細 | 補足説明 |
