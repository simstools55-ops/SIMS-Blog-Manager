# 日次運用

## 1日の最初の起動

その日に初めてスプレッドシートを開き、まだ日次更新を実行していない場合だけ確認画面が表示されます。

- **更新する**：Search Consoleの最新ページデータを取得し、記事DBと今日の改善を更新します。
- **今回は更新しない**：画面を閉じます。手動更新は「SIMS-Blog-Manager → 日次処理を実行」から実行できます。

当日すでに更新済みの場合、起動時の確認画面は表示されません。

## 日次更新で更新する情報

- クリック数
- 表示回数
- CTR
- 掲載順位
- 記事ランク
- 今日の改善
- Homeの集計

記事タイトル、SEOタイトル、メタディスクリプション、メインクエリ、ArticleID、改善履歴、作業状態は保持します。

## Search Consoleに出なかった記事

その日の取得結果にURLがなくても、記事DBから削除しません。

- 1～2回連続で未取得：**データ未取得**
- 3回連続、または最終確認から14日以上：**要確認**

「データ未取得」と「要確認」の記事は、通常の改善候補から除外されます。記事URL、インデックス状況、noindex、リダイレクト、削除の有無を利用者が確認してください。

確認が終わるまでは、記事DBの過去データを保持します。

## 新規記事が見つかった場合

日次更新後に新規記事が見つかると、記事情報を取得するか確認する画面が表示されます。

取得対象：

- 記事タイトル
- SEOタイトル
- メタディスクリプション
- メインクエリ

## 改善完了の登録

記事の修正が終わったら、改善ナビ下部の **「改善完了を登録」** を押します。Claudeの回答末尾にある `SIMS_FEEDBACK_V1` のJSONを貼り付けて登録してください。

登録後は記事管理が「モニター中」へ移り、「今日の改善」の対象から外れます。
