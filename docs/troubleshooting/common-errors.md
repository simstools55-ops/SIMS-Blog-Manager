# よくあるエラー

## 改善履歴の書式やチェックボックスが反映されない

1. 最新の`コード.gs`へ全置換します。
2. スプレッドシートを再読み込みします。
3. **SIMS-Blog-Manager → シートの作成・修復**を1回実行します。
4. **改善履歴 → 改善履歴を開く**を実行します。

Product 5.2.0では、改善履歴を開くたびに一覧、書式、チェックボックスを再反映します。

## 日次更新の確認画面をスキップした

**SIMS-Blog-Manager → 日次処理を実行**から再実行できます。

## 今日の改善が空

日次処理を実行し、記事管理にSearch Consoleデータがあることを確認します。改善候補がない場合は、候補条件に合う記事がない可能性があります。

## 未取得記事が表示される

Search Consoleから一時的にデータを取得できなかった記事などです。記事が削除されたことを意味しません。URL、インデックス、noindex、リダイレクトの有無を確認してください。

## バージョン確認

**SIMS-Blog-Manager → バージョン情報**で`5.2.0`と表示されることを確認します。
