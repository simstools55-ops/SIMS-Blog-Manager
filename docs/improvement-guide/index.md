# 改善ガイド

このページでは、改善ナビを見た後に何を直すかの考え方をまとめます。

## titleタグ修正

CTRが低く、順位が比較的高い記事で有効です。メインクエリを自然に含め、検索者がクリックしたくなる具体性を加えます。

## 冒頭文修正

検索意図と記事内容の一致を早く伝えたい場合に有効です。最初の数行で「この記事で分かること」を明確にします。

## H2追加・修正

順位が11〜20位付近の記事で有効です。参考クエリを使い、検索者が知りたい項目を見出し化します。

## FAQ追加

似た質問クエリが多い場合に有効です。FAQ候補をもとに、短く答えるセクションを追加します。

## 内部リンク追加

Product 5.2では、改善ナビを開くと記事DBとSearch Consoleのクエリデータから、関連性の高いブログ内記事を最大8件抽出します。

候補には次の情報が表示され、AIへの依頼文にも自動で追加されます。

- 記事タイトル
- 記事URL
- メインクエリ
- 関連すると判断した理由
- 検索意図が近すぎる場合の注意

候補はすべて使う必要はありません。本文の流れと読者の検索意図に合う記事だけを選び、自然な位置とアンカーテキストを提案するようAIへ依頼します。十分な関連性がない場合は、件数を満たすためだけに無関係な記事を追加しません。
