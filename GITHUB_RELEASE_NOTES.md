# Product 5.5.2

## 日次処理の状態確認多重実行を修正

Product 5.5.1では、日次処理中に2.5秒間隔の状態確認を固定的に投入していました。1回の状態確認が2.5秒を超える環境では確認処理が重なり、Apps Scriptの実行履歴が `sbmGetDailyUpdateClientStatus` で埋まる問題がありました。

Product 5.5.2では、前回の状態確認が完了してから10秒後に次の確認を1件だけ実行します。

- 状態確認の同時実行を禁止
- `setInterval` を廃止し、応答完了後の `setTimeout` に変更
- ダイアログ終了時にポーリングを停止
- 実行ボタンの二重起動を防止
- 重複していたロック取得を修正

日次処理本体、Search Console連携、既存シート構成、権限スコープは変更していません。
